# < TreeTraverser /> ⚙️

El componente <span>Tree Traverser</span> permite la navegación por distintas secciones utilizando 
una estructura de árbol para llegar desde el nodo padre hasta el nodo hijo.

<br />

## Propiedades

El componente <span>Tree Traverser</span> puede recibir las siguientes propiedades:
<ul>
    <li><span>data :</span> Objeto que contiene el volumen de datos a renderizar. 
    Observar los detalles de esta *prop* en la sección **Data**</li>
    <li>fatherProp: Patrón mediante el cual se realiza la navegación por el árbol</li>
    <li>set: Función que actualiza el estado correspondiente de la función *set* pasada</li>
    <li>validate: Función definida con el propósito de desplegar el componente</li>
    <li>invalidate: Función definida con el propósito de ocultar el componente</li>
  </ul>
Observar que las *props* destacadas en negrita son obligatorias para el correcto funcionamiento del componente.

<br />

### Data

El componente tiene la *prop* obligatoria **data** que contiene el volumen de datos a renderizar. 

<span>Data</span> es un arreglo de tipo `TreeStructure` que cuenta con la siguiente interfaz:

```
    interface TreeStructure {
      [key: string]: string | number | boolean | undefined | never[];
      id: string;
      description: string;
    }
```

Donde tiene definidas las propiedades `id` y `description`. 

La notación `[key: string]: string | number | boolean | undefined | never[];`
permite agregar de manera opcional propiedades donde la clave es de tipo `string` y su valor puede ser cualquiera de los que se muestra
a continuación.

<br />

### fatherProp

La *prop* opcional *fatherProp* determina el patrón mediante el cual se va a realizar la navegación. 

Por ejemplo, si se determina un volumen de datos con productos y se asigna `fatherProp='fatherId'`, 
la navegación por el árbol se va realizar teniendo en cuenta si, el `id` del producto seleccionado coincide 
con el valor de `fatherId` de cualquier otro producto, entonces se van a listar todos aquellos donde haya 
coincidencia hasta seleccionar un producto que no tenga la propiedad `fatherId`. 

<br /> 

<span>NOTA:</span> En un ejemplo que no sea de modelo, se recomienda fuertemente la práctica del *hook* `useContext` 
en lugar de utilizar estados locales y compartirlos mediante *props*; y de la libreria <a href='https://redux-toolkit.js.org/' target='blank'>Redux</a> para el manejo de estados y acciones globales.
