# < DynamicForm /> ⚙️

<br /> 

## Formulario dinámico

El componente DynamicForm es el que usamos para todos los formularios, no importa su nivel de complejidad. Recibe un JSONSchema como valor de la propiedad 'json' y a partir del mismo renderea los inputs necesarios y genera las reglas de validación. Si el valor de la propiedad json es falsy muestra un spinner, si se le pasa un objecto vacío (o cualquier cosa que no sea un JSONSchema) muestra un mensaje de error.

```
import FORM_JSON from '../utils/json-schema-form.json'

<DynamicForm json={FORM_JSON} handleSubmit={ (values) => console.log(values) } />
```

<span>Este componenente no tiene un tamaño propio, está hecho para adaptarse a un contenedor.</span>

Una especificación sobre el objeto que recibe este componente para crear el formulario puede verse [aquí](https://drive.google.com/file/d/1-0CrTLkvraYF7AIoCQ_mB8zvzgBXCzWh/view?usp=sharing), incluyendo una descripción detallada de cada parte del JSONSchema y un repositorio de ejemplos de DTOs válidos.
<br/>

### Formulario sin validaciones

Si no hace falta realizar validaciones a los inputs de un formulario, podemos hacer uso de la propiedad 'validate' del componente. Esta propiedad tiene como valor por defecto 'true'.

```
import NO_VALIDATIONS_JSON from '../utils/json-schema-form.json'

<DynamicForm json={NO_VALIDATIONS_JSON} handleSubmit={} validate={ false } />
```
